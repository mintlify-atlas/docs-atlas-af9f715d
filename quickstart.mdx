Get your first gender prediction model up and running in minutes. This guide walks you through installation, training, and making predictions.

## Prerequisites

Before you begin, ensure you have:
- Python 3.8 or higher installed
- Your training data file: `boy-or-girl-2025-train-missingValue.csv`
- Your test data file: `boy-or-girl-2025-test-no-ans-missingValue.csv`

<Note>
This project uses scikit-learn, XGBoost, and pandas for machine learning. All dependencies will be installed in the next step.
</Note>

## Installation

<Steps>
  <Step title="Clone the repository">
    First, clone the project to your local machine:
    
    ```bash
    git clone https://github.com/yourusername/IM5033-boy-or-girl-2026.git
    cd IM5033-boy-or-girl-2026
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required packages from `requirements.txt`:
    
    ```bash
    pip install -r requirements.txt
    ```
    
    This installs the following key packages:
    - **pandas** (3.0.1) - Data manipulation
    - **scikit-learn** (1.8.0) - Machine learning models
    - **xgboost** (3.2.0) - Gradient boosting
    - **numpy** (2.4.2) - Numerical operations
    - **matplotlib** (3.10.8) & **seaborn** (0.13.2) - Visualization
    
    <Tip>
    Consider using a virtual environment to keep dependencies isolated:
    ```bash
    python -m venv venv
    source venv/bin/activate  # On Windows: venv\Scripts\activate
    pip install -r requirements.txt
    ```
    </Tip>
  </Step>

  <Step title="Prepare your data">
    Place your training and test data files in a `data/` directory:
    
    ```bash
    mkdir -p data
    # Copy your CSV files to the data directory
    ```
    
    Expected data structure:
    ```
    IM5033-boy-or-girl-2026/
    ├── data/
    │   ├── boy-or-girl-2025-train-missingValue.csv
    │   └── boy-or-girl-2025-test-no-ans-missingValue.csv
    ├── train_model.py
    ├── predict.py
    └── data_pipeline.py
    ```
    
    <Warning>
    Update the `root_dir` variable in `train_model.py` and `predict.py` to match your local path:
    ```python
    root_dir = "/path/to/your/IM5033-boy-or-girl-2026"
    ```
    </Warning>
  </Step>
</Steps>

## Training Your First Model

<Steps>
  <Step title="Run the training script">
    Train models and select the best performer:
    
    ```bash
    python train_model.py
    ```
    
    The script will:
    1. Load and clean the training data using `data_pipeline.clean_data()`
    2. Handle missing values and outliers automatically
    3. Train three models: Logistic Regression, Random Forest, and XGBoost
    4. Evaluate each model on validation data
    5. Save the best model based on ROC-AUC score
  </Step>

  <Step title="Review the results">
    You'll see output similar to this:

    <Note>
    The model automatically handles class imbalance using `class_weight='balanced'` for scikit-learn models and `scale_pos_weight` for XGBoost.
    </Note>
  </Step>

  <Step title="Understand the preprocessing">
    The data pipeline (`data_pipeline.py`) performs these operations:
    
    **Data Cleaning:**
    - Converts numeric columns with errors (like `#NUM!`) to NaN
    - Removes extreme outliers (values > 1e12 or < -1e12)
    - Applies domain-specific validation:
      - Height: 100-250 cm range
      - Weight: 30-200 kg range
      - YouTube views: Must be non-negative
    
    **Feature Engineering:**
    ```python
    # Creates a new feature from self-introduction length
    df['self_intro_len'] = df['self_intro'].fillna('').astype(str).apply(len)
    ```
    
    **Preprocessing Pipeline:**
    - **Numeric features** (height, weight, sleepiness, iq, fb_friends, yt, self_intro_len):
      - Median imputation for missing values
      - Standard scaling (zero mean, unit variance)
    - **Categorical features** (star_sign, phone_os):
      - Unknown imputation for missing values
      - One-hot encoding
  </Step>
</Steps>

## Making Predictions

<Steps>
  <Step title="Run the prediction script">
    Generate predictions for your test data:
    
    ```bash
    python predict.py
    ```
    
    This will:
    1. Load the saved best model and preprocessor
    2. Clean and preprocess test data using the same pipeline
    3. Generate predictions
    4. Create a submission file
  </Step>

  <Step title="Get your submission file">
    Find your predictions in `submission.csv`:
    
    ```plaintext
    Predictions saved to /path/to/submission.csv
    ```
    
    The submission file format:
    ```csv
    id,gender
    1,1
    2,2
    3,1
    4,2
    ...
    ```
    
    Where gender values are:
    - `1` = Male
    - `2` = Female
    
    <Tip>
    This CSV file is ready to submit directly to Kaggle competitions using the standard submission format.
    </Tip>
  </Step>
</Steps>

## Understanding the Features

The model uses these features to make predictions:

| Feature | Type | Description |
|---------|------|-------------|
| `height` | Numeric | Height in centimeters (100-250 range) |
| `weight` | Numeric | Weight in kilograms (30-200 range) |
| `sleepiness` | Numeric | Sleepiness score |
| `iq` | Numeric | IQ score |
| `fb_friends` | Numeric | Number of Facebook friends |
| `yt` | Numeric | YouTube-related metric (non-negative) |
| `self_intro_len` | Numeric | Length of self-introduction text (engineered) |
| `star_sign` | Categorical | Astrological star sign |
| `phone_os` | Categorical | Phone operating system |
