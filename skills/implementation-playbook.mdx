## 1. Python Engineering Standards

<Accordion title="Structure & Style">

### PEP 8 Compliance

All code must strictly adhere to [PEP 8](https://pep8.org/) style guide:
- 4 spaces for indentation (no tabs)
- Maximum line length of 79 characters for code
- Maximum line length of 72 characters for docstrings
- Two blank lines between top-level definitions
- One blank line between method definitions

Use Python type hints (`typing` module) for all function signatures:

<Accordion title="Error Handling">

### No Bare Except

Never use bare `except:` clauses:

```python
# Bad
try:
    result = risky_operation()
except:
    print("Something went wrong")

# Good
try:
    result = risky_operation()
except ValueError as e:
    logger.error(f"Invalid value: {e}")
    raise
except FileNotFoundError as e:
    logger.error(f"File not found: {e}")
    return None
```

### Custom Exceptions

Use custom exception classes for domain-specific errors:

```python
class DataValidationError(Exception):
    """Raised when data validation fails."""
    pass

class ModelNotTrainedError(Exception):
    """Raised when attempting to predict with untrained model."""
    pass

def validate_schema(df: pd.DataFrame, expected_columns: List[str]) -> None:
    missing = set(expected_columns) - set(df.columns)
    if missing:
        raise DataValidationError(f"Missing columns: {missing}")
```

### Logging

Implement logging using the `logging` library. **Avoid `print()` in production-ready scripts**:

```python
import logging

# Setup logging
logging.basicConfig(
    level=logging.INFO,
    format='%(asctime)s - %(name)s - %(levelname)s - %(message)s',
    handlers=[
        logging.FileHandler('app.log'),
        logging.StreamHandler()
    ]
)

logger = logging.getLogger(__name__)

# Usage
logger.info("Starting training process")
logger.warning(f"Missing values detected: {null_count}")
logger.error(f"Training failed: {error}")
```

## 2. Data Science Workflow

<Accordion title="Data Integrity">

### Immutability

**Never overwrite raw data.** Always version processed datasets:

```
data/
├── raw/
│   └── train.csv (never modified)
├── interim/
│   └── train_cleaned.csv
└── processed/
    ├── v1.parquet
    ├── v2.parquet
    └── v3.parquet
```

```python
# Bad
df = pd.read_csv('data/train.csv')
df = preprocess(df)
df.to_csv('data/train.csv')  # Overwrites original!

# Good
df = pd.read_csv('data/raw/train.csv')
df_processed = preprocess(df)
df_processed.to_parquet('data/processed/v1.parquet')
```

### Validation

Use `pandera` or basic `assert` statements to verify schema and data types after every transformation:

```python
import pandera as pa
from pandera import Column, Check

# Define schema
schema = pa.DataFrameSchema({
    "age": Column(int, Check.in_range(0, 120)),
    "hair_length": Column(float, Check.greater_than(0)),
    "favorite_color": Column(str),
})

# Validate
df_validated = schema.validate(df)

# Or use assertions
assert df['age'].min() >= 0, "Age has negative values"
assert df['age'].max() <= 120, "Age has unrealistic values"
assert df['hair_length'].isnull().sum() == 0, "hair_length has missing values"
```

### Efficiency

Prefer `polars` or **vectorized pandas** over manual loops:

```python
# Bad: Loop
for i in range(len(df)):
    df.loc[i, 'bmi'] = df.loc[i, 'weight'] / (df.loc[i, 'height'] ** 2)

# Good: Vectorized
df['bmi'] = df['weight'] / (df['height'] ** 2)

# Better: Polars (faster for large datasets)
import polars as pl
df_polars = pl.DataFrame(df)
df_polars = df_polars.with_columns(
    (pl.col('weight') / (pl.col('height') ** 2)).alias('bmi')
)
```

## 3. Machine Learning Excellence

<Accordion title="Experiment Tracking">

All training runs must be logged. If no remote server exists, log to a local `experiments.csv`.

### Mandatory Metadata:
1. Model version
2. Hyperparameters
3. Git hash (commit ID)
4. Hardware used (CPU/GPU)
5. Training time
6. Dataset version
7. Metrics achieved

```python
import subprocess
import datetime
import pandas as pd

def log_experiment(params: dict, metrics: dict) -> None:
    """Log experiment to local CSV."""
    # Get git hash
    git_hash = subprocess.check_output(['git', 'rev-parse', 'HEAD']).decode().strip()
    
    log_entry = {
        'timestamp': datetime.datetime.now().isoformat(),
        'git_hash': git_hash,
        'model_version': params.get('model_version'),
        **params,
        **metrics
    }
    
    df = pd.DataFrame([log_entry])
    df.to_csv('experiments.csv', mode='a', header=False, index=False)
```

</Accordion>

<Accordion title="Model Evaluation">

### Baselines

A **"Dummy Regressor/Classifier" baseline** must be established before training complex models:

```python
from sklearn.dummy import DummyClassifier, DummyRegressor
from sklearn.metrics import f1_score

# Classification baseline
baseline = DummyClassifier(strategy='most_frequent')
baseline.fit(X_train, y_train)
baseline_score = f1_score(y_test, baseline.predict(X_test))

print(f"Baseline F1: {baseline_score:.4f}")
print("Your model must beat this!")
```

### Metrics

#### Classification:

**Precision-Recall curves are preferred over ROC** for imbalanced data:

```python
from sklearn.metrics import precision_recall_curve, average_precision_score
import matplotlib.pyplot as plt

precision, recall, _ = precision_recall_curve(y_test, y_proba)
ap_score = average_precision_score(y_test, y_proba)

plt.plot(recall, precision)
plt.xlabel('Recall')
plt.ylabel('Precision')
plt.title(f'Precision-Recall Curve (AP={ap_score:.3f})')
plt.savefig('reports/pr_curve.png')
```

#### Regression:

Include **MAE and Max Error** alongside RMSE:

```python
from sklearn.metrics import mean_absolute_error, mean_squared_error, max_error
import numpy as np

mae = mean_absolute_error(y_test, y_pred)
rmse = np.sqrt(mean_squared_error(y_test, y_pred))
max_err = max_error(y_test, y_pred)

print(f"MAE: {mae:.4f}")
print(f"RMSE: {rmse:.4f}")
print(f"Max Error: {max_err:.4f}")
```

## 4. MLOps & Deployment

<Accordion title="Reproducibility">

### Environment

All projects must include a `requirements.txt` or `pyproject.toml`:

```bash
# Generate requirements.txt
pip freeze > requirements.txt

# Or use pipreqs (only actual imports)
pipreqs . --force
```

### Seeding

Global seeds (`numpy`, `torch`, `random`) must be set to `42` unless otherwise specified:

```python
import random
import numpy as np
import torch

SEED = 42

random.seed(SEED)
np.random.seed(SEED)
torch.manual_seed(SEED)
if torch.cuda.is_available():
    torch.cuda.manual_seed_all(SEED)

# For complete reproducibility in PyTorch
torch.backends.cudnn.deterministic = True
torch.backends.cudnn.benchmark = False
```

</Accordion>

<Accordion title="Serialization">

### Scikit-Learn

Use **`joblib`** for Scikit-Learn models:

```python
import joblib

# Save
joblib.dump(model, 'models/model_v1.pkl')

# Load
model = joblib.load('models/model_v1.pkl')
```

### Deep Learning

Use **`safetensors`** for Deep Learning weights:

```python
from safetensors.torch import save_file, load_file
import torch

# Save
save_file(model.state_dict(), 'models/model_v1.safetensors')

# Load
state_dict = load_file('models/model_v1.safetensors')
model.load_state_dict(state_dict)
```

### Metadata

Include a **`metadata.json`** alongside every saved model:

```python
import json
from datetime import datetime

metadata = {
    'model_type': 'XGBClassifier',
    'version': 'v1.0',
    'created_at': datetime.now().isoformat(),
    'features': list(X_train.columns),
    'performance': {
        'f1_score': 0.87,
        'accuracy': 0.85
    },
    'hyperparameters': {
        'n_estimators': 100,
        'max_depth': 5,
        'learning_rate': 0.1
    }
}

with open('models/model_v1_metadata.json', 'w') as f:
    json.dump(metadata, f, indent=2)
```

</Accordion>

## 5. Verification Checklist (The "Final Gate")

Before declaring a task "Complete," verify:

<Accordion title="Pre-Completion Checklist">

- [ ] **Code Formatting**: Passes `flake8` or `black` formatting
  ```bash
  black .
  flake8 . --max-line-length=88
  ```

- [ ] **No Data Leakage**: Training data does not contain information from the future/test set
  - Check timestamp splits
  - Verify train/test split happens before preprocessing
  - Confirm no target leakage in features

- [ ] **Baseline Comparison**: Model performance beats the established baseline
  ```python
  assert model_score > baseline_score, "Model doesn't beat baseline!"
  ```

- [ ] **Documentation**: Includes a "How to Run" section
  - Example commands to reproduce results
  - Environment setup instructions
  - Expected outputs

- [ ] **Type Hints**: All functions have type annotations

- [ ] **Logging**: Uses `logging` library, not `print()`

- [ ] **Error Handling**: No bare `except:` clauses

- [ ] **Tests**: Critical functions have unit tests (if applicable)

- [ ] **Reproducibility**: Seeds are set, requirements documented

- [ ] **Data Validation**: Schema validation or assertions in place

</Accordion>
